## 1. 什么是养马测试软件

**养马测试软件** 是一套专为 [Hermes Agent](https://hermes-agent.nousresearch.com) 设计的调教质量评估系统。

> 「养马」= 驯养良驹 = 调教、优化 Hermes Agent

它通过自动扫描 `~/.hermes/` 环境（技能、配置文件、Cron 任务、外部工具、安全规则等），从六个维度量化评估 Agent 的配置水平，计算 **HTS（Hermes Tuning Score）** 综合评分，并生成改进建议。

### 核心特性

| 特性 | 说明 |
|------|------|
| 🎯 **量化评估** | 六维评分模型 + 加权几何平均算法，0-1000 分制 |
| 🔍 **诊断短板** | 自动检测技能覆盖盲区、配置缺陷、安全遗漏 |
| 📈 **趋势追踪** | JSON 历史记录，支持评分趋势对比 |
| 💡 **改进建议** | 针对低分维度自动生成可操作建议 |
| 📄 **Word 报告** | 一键生成桌面 Word 详细报告 |
| 🌐 **跨平台** | macOS / Windows / Linux 均可运行 |
| 🚫 **零依赖** | 纯 Python 标准库，无需 `pip install` |

## 5. 评分模型

### 5.1 六维评分体系

| # | 维度 | 权重 | 子项数 | 评估内容 |
|---|------|------|--------|----------|
| 🧠 | **技能体系** (Skills) | **25%** | 5 | 技能数量、frontmatter 质量、描述结构化、领域覆盖、社区利用 |
| ⚙️ | **配置调优** (Config) | **15%** | 6 | Provider、Fallback、Profile、Token 优化、记忆、网络 |
| 📝 | **Prompt 工程** (Prompt) | **15%** | 5 | System Prompt 质量、Skill 描述、上下文管理、多语言、指令精确度 |
| 🔌 | **工具生态** (Tools) | **20%** | 5 | 插件、Gateway、Cron 任务、外部工具、API |
| 📊 | **运营水平** (Operations) | **10%** | 5 | 记忆质量、迭代优化、社区参与、文档化、自动化 |
| 🛡️ | **安全隐私** (Security) | **15%** | 5 | 隐私规则、规则质量、跨 Profile 保护、数据隔离、安全意识 |

### 5.2 HTS 算法

**加权几何平均**（Weighted Geometric Mean）：

```
HTS_base = 1000 × S₁^W₁ × S₂^W₂ × ... × Sₙ^Wₙ

HTS_final = HTS_base × β × σ

其中：
  Sᵢ = 第 i 维度的得分 (0.0 ~ 1.0)
  Wᵢ = 第 i 维度的权重 (∑Wᵢ = 1.0)
  β  = 趋势修正系数（上升 +5%，下降 -5%）
  σ  = 稳定性修正系数（预留，当前 = 1.0）
```

**为什么用几何平均？**
- 几何平均对低分维度更敏感，避免「一俊遮百丑」
- 安全隐私等短板不容忽视，几何平均能真实反映

### 5.3 评分客观性

v4.0 不再使用固定分或关键词存在性检查，而是：

| 子项 | v1.0 方式 | v4.0 方式 | 提升 |
|------|-----------|-----------|------|
| `skill_desc` | 检查 description/usage 关键词 | **frontmatter 字段完整度 + 描述结构化分析** | 从「有没有」到「好不好」 |
| `cfg_token` | 固定 10 分 | **实际检测 max_tokens / context_window** | 从「假设」到「事实」 |
| `sec_cross_profile` | 固定 15 分 | **检测 config.yaml 实际配置** | 从「假设」到「事实」 |
| `sec_data_isolate` | 固定 12 分 | **基于 .hermes.md 四维深度分析** | 从「假设」到「事实」 |

## 9. 常见问题

### Q1: 为什么我的 HTS 比算术平均低很多？

因为评分使用**加权几何平均**，对低分维度有惩罚效应。如果某个维度得分很低，即使其他维度满分，HTS 也会被大幅拉低。这不是 bug——几何平均更能反映真实水平。

### Q2: 需要装什么依赖才能用？

**终端版零依赖。** `python-docx` 仅用于生成 Word 报告，是可选项。`PyYAML` 不需要——v4.0 用纯正则表达式解析 frontmatter。

### Q3: 如何提高技能描述分数？

确保每个 SKILL.md 包含：
1. ✅ 完整的 YAML frontmatter（name / description / version / author / license / metadata）
2. ✅ Usage 章节（使用说明）
3. ✅ Parameters 章节（参数说明）
4. ✅ Examples 章节（示例代码）
5. ✅ Dependencies 章节（依赖说明）

### Q4: 跨 Profile 保护分数怎么拿满？

在 `~/.hermes/config.yaml` 中配置 `cross_profile`，确保不同 Profile 之间的数据隔离。

### Q5: Word 报告生成失败？

```bash
pip3 install python-docx lxml
```

然后重新运行 `python3 horse_test.py`。

### Q6: Windows 上 cmd 显示乱码？

终端彩色输出在 Windows 命令提示符中可能无法正确显示。不影响评分功能，文字信息仍然可读。建议使用 Windows Terminal 或 VS Code 终端。

### Q7: 提示 FileNotFoundError: pgrep？

Windows 系统没有 `pgrep` 命令。v4.0 已内置跨平台支持，会自动切换到 `tasklist`。如果还有问题，请确保已更新到最新版本。
